# Surface 不同国家 Wi-Fi Calling 规则草案

## 1. 目标

为 Surface 设备在不同国家/地区提供统一的 Wi-Fi Calling 开关与可用性判断规则，确保：

- 用户看到的能力与当地法规、运营商能力一致
- 设备在漫游、出境、跨区使用时行为一致
- 客服、产品、研发可以用同一套口径排查问题

## 2. 适用范围

- 适用于 Surface 设备上的蜂窝语音能力
- 适用于 Wi-Fi Calling 的开关展示、激活、呼叫路由和错误提示
- 适用于“国家/地区 + 运营商 + 设备版本 + SIM/eSIM 状态”的组合判断

## 3. 基础原则

1. 以“国家/地区 + 运营商策略”为准，不以设备当前连接的 Wi-Fi 地点单独决定
2. 只有在“当地法规允许 + 运营商支持 + 机型支持 + 账号/套餐支持”同时满足时才可启用
3. 默认策略采用“保守可控”原则，先不展示，再按配置放开
4. 漫游时不默认继承母国规则，必须单独判断当前 SIM 所属市场和本地网络政策

## 4. 规则分层

### 4.1 国家/地区层

将国家/地区分为三类：

- `A 类：允许`
  - 当地法规允许 Wi-Fi Calling
  - 可正常展示开关
  - 满足运营商条件后可启用
- `B 类：条件允许`
  - 当地法规允许，但需要按运营商/套餐/认证白名单控制
  - 默认隐藏或灰置，依据配置放量
- `C 类：禁止`
  - 当地法规不允许，或明确要求关闭
  - 不展示开关，不允许激活

### 4.2 运营商层

同一国家内，不同运营商可能策略不同，规则优先级如下：

1. 监管/法律限制
2. 运营商白名单
3. 套餐/账号资格
4. 设备型号与软件版本
5. 用户侧设置

### 4.3 设备层

设备必须同时满足：

- 支持 Wi-Fi Calling 的硬件/基带能力
- 已通过对应市场认证
- 固件/系统版本满足最低版本要求
- 设备未被策略禁用

## 5. 决策逻辑

### 5.1 展示规则

是否展示 Wi-Fi Calling 开关：

```text
showToggle =
  countryStatus != "C"
  AND carrierSupported == true
  AND deviceSupported == true
  AND accountEligible == true
```

### 5.2 可用规则

是否允许开启并成功注册：

```text
enabled =
  showToggle == true
  AND simPresent == true
  AND imsProvisioned == true
  AND emergencyPolicyAllowed == true
```

### 5.3 呼叫路由规则

- Wi-Fi Calling 已启用且注册成功：优先走 Wi-Fi 语音
- Wi-Fi 信号不可用或 IMS 失配：自动回落到蜂窝网络
- 在禁止国家/地区：即使连接 Wi-Fi，也必须回落到普通蜂窝语音或直接关闭相关入口

## 6. 国家规则模板

可按下面表格维护各国家/地区配置：

| 国家/地区 | 分类 | 默认展示 | 默认可用 | 备注 |
|---|---|---:|---:|---|
| 国家 A | A 类 | 是 | 是 | 运营商白名单放开 |
| 国家 B | B 类 | 视运营商而定 | 视运营商而定 | 需单独认证 |
| 国家 C | C 类 | 否 | 否 | 法规禁止 |

## 7. 用户文案建议

### 7.1 未支持

- “当前所在国家/地区或运营商不支持 Wi-Fi Calling。”

### 7.2 条件不足

- “你的号码套餐或运营商暂不支持 Wi-Fi Calling。”

### 7.3 已启用但不可注册

- “Wi-Fi Calling 已开启，但当前网络环境无法完成注册，请稍后重试。”

## 8. 例外处理

1. 监管要求临时调整时，国家级配置优先级最高，可直接关闭
2. 紧急呼叫相关规则必须单独校验，不得因 Wi-Fi Calling 关闭影响基础紧急通话能力
3. 如果运营商要求仅对特定 SKU、IMEI 段或软件版本开放，需支持更细粒度白名单

## 9. 推荐的配置字段

- `country_code`
- `carrier_id`
- `sim_mcc_mnc`
- `device_model`
- `os_version_min`
- `country_status`
- `carrier_wifi_calling_enabled`
- `account_eligible`
- `emergency_policy_allowed`

## 10. 落地建议

- 采用配置中心管理国家/运营商白名单
- 对外层展示与底层注册分开控制
- 增加埋点：
  - 国家/运营商命中情况
  - 开关展示结果
  - 注册成功/失败原因
  - 回落到蜂窝网络的原因

## 11. 可直接复用的简版规则

> 当 `国家/地区允许` 且 `运营商支持` 且 `设备支持` 且 `账号/套餐允许` 时，Surface 才展示并允许开启 Wi-Fi Calling；否则隐藏或禁用。禁止国家/地区一律关闭。漫游场景按当前 SIM 所属市场与运营商规则重新判断。
